Documentación de API - Inventario Facilito


Usuarios

Descripción

Registra un nuevo usuario en la base de datos con el nombre de usuario, contraseña y correo electrónico proporcionados.

Parámetros
  • Usuario: Nombre de usuario.
  • Contrasena: Contraseña del usuario.
  • Correo: Correo electrónico del usuario.
Ejemplo de uso
        
POST /Registrar
Content-Type: application/json

{
  "Usuario": "nuevoUsuario",
  "Contrasena": "password123",
  "Correo": "correo@ejemplo.com"
}
Respuestas
  • 200 OK: Si el registro es exitoso. Retorna un mensaje indicando que el usuario ha sido creado.
  • 409 Conflict: Si el nombre de usuario ya existe. Retorna un mensaje indicando que el usuario ya existe.
  • 500 Internal Server Error: Error interno del servidor.
Descripción

Autentica a un usuario verificando su correo y contraseña. Retorna los detalles del usuario si la autenticación es exitosa.

Parámetros
  • Correo: Correo electrónico del usuario (en el cuerpo de la solicitud).
  • Contrasena: Contraseña del usuario (en el cuerpo de la solicitud).
Ejemplo de Uso
        
POST /Login
Content-Type: application/json

{
  "Correo": "juan@mail.com",
  "Contrasena": "1234"
}
Respuesta Exitosa (200 OK)
        
{
  "Usuario": "juan@mail.com",
  "Contrasena": "1234"
}
Respuestas
  • 200 OK: Retorna los detalles del usuario si la autenticación es exitosa.
  • 401 Unauthorized: Retorna un error si las credenciales proporcionadas son incorrectas.
  • 500 Internal Server Error: Retorna un error si ocurre un problema al procesar la solicitud.
Descripción

Obtiene el código de administrador asociado al correo electrónico proporcionado.

Parámetros
  • Correo_usuario: Correo electrónico del usuario.
Ejemplo de uso
        
GET /Obtener_Codigo_Administrador?Correo_usuario=correo@ejemplo.com
Respuestas
  • 200 OK: Si la consulta es exitosa. Retorna el código de administrador asociado al correo proporcionado.
  • 400 Bad Request: Si no se proporciona el parámetro Correo_usuario. Retorna un mensaje indicando que el parámetro es requerido.
  • 404 Not Found: Si no se encuentra un usuario con el correo proporcionado. Retorna un mensaje indicando que el usuario no fue encontrado.
  • 500 Internal Server Error: Error interno del servidor.

Bodegas

Descripción

Obtiene la lista de bodegas administradas por el usuario asociado al correo electrónico proporcionado, incluyendo información sobre ubicación y etiquetas asociadas.

Parámetros
  • Correo_usuario: Correo electrónico del usuario.
Ejemplo de uso
        
GET /Obtener_Bodegas?Correo_usuario=correo@ejemplo.com
Respuestas
  • 200 OK: Si la consulta es exitosa. Retorna la lista de bodegas administradas por el usuario, incluyendo información de ubicación y etiquetas.
  • 401 Unauthorized: Si hay un error en la consulta o si el usuario no tiene permisos. Retorna un mensaje indicando que el usuario es incorrecto.
  • 500 Internal Server Error: Error interno del servidor.
Descripción

Obtiene la lista de bodegas donde el usuario invitado trabaja como empleado, incluyendo información sobre ubicación, etiquetas y código de administrador.

Parámetros
  • Correo_usuario: Correo electrónico del usuario invitado.
Ejemplo de uso
        
GET /Obtener_Bodegas_Invitado?Correo_usuario=correo@ejemplo.com
Respuestas
  • 200 OK: Si la consulta es exitosa. Retorna la lista de bodegas donde el usuario invitado trabaja como empleado, incluyendo información de ubicación, etiquetas y código de administrador si aplica.
  • 400 Bad Request: Si no se proporciona el parámetro Correo_usuario. Retorna un mensaje indicando que el parámetro es requerido.
  • 500 Internal Server Error: Error interno del servidor.
Descripción

Crea una nueva bodega y la asocia con el usuario especificado. También se asocian las etiquetas proporcionadas con la bodega.

Parámetros
  • Nombre_bodega: Nombre de la nueva bodega.
  • Correo_usuario: Correo electrónico del usuario que administrará la bodega.
  • Lista_tags: Array de IDs de etiquetas a asociar con la bodega.
  • UbicacionNombre: Nombre de la ubicación de la bodega.
  • UbicacionLatitud: Latitud de la ubicación de la bodega.
  • UbicacionLongitud: Longitud de la ubicación de la bodega.
Ejemplo de uso
        
POST /Crear_bodega
Content-Type: application/json

{
  "Nombre_bodega": "Bodega Central",
  "Correo_usuario": "usuario@ejemplo.com",
  "Lista_tags": [1, 2, 3],
  "UbicacionNombre": "Centro de la Ciudad",
  "UbicacionLatitud": 40.712776,
  "UbicacionLongitud": -74.005974
}
Respuestas
  • 200 OK: Si la bodega se crea y asocia correctamente. Retorna un mensaje indicando que la bodega ha sido creada y asociada correctamente.
  • 400 Bad Request: Si faltan parámetros o si los datos proporcionados son incorrectos. Retorna un mensaje indicando que los datos son insuficientes o incorrectos.
  • 401 Unauthorized: Si el usuario especificado no existe. Retorna un mensaje indicando que el usuario es incorrecto.
  • 500 Internal Server Error: Error interno del servidor en la creación de la bodega o en la asociación de etiquetas.

Venta

Descripción

Realiza una venta de productos desde una bodega específica, registrando los detalles de la venta en la base de datos. El proceso incluye:

  • Creación de un registro de boleta para la venta.
  • Actualización del stock de productos en la bodega según la cantidad vendida.
  • Registro de cada compra y sus detalles en la boleta.
Parámetros
  • clienteId: ID del cliente que realiza la compra.
  • items: Array de objetos que contiene los siguientes campos:
    • productoId: ID del producto a vender.
    • cantidad: Cantidad del producto a vender.
  • correoUsuario: Correo electrónico del usuario que realiza la venta. Este parámetro se usa para obtener el Usuario_ID.
  • bodegaId: ID de la bodega desde la cual se realiza la venta.
Ejemplo de Uso
        
POST /vender
Content-Type: application/json

{
  "clienteId": 1,
  "items": [
    {"productoId": 1, "cantidad": 3},
    {"productoId": 2, "cantidad": 1}
  ],
  "correoUsuario": "juan@mail.com",
  "bodegaId": 2
}
Respuestas
  • 200 OK: Si la venta se realiza correctamente. Retorna un mensaje indicando que la venta se completó exitosamente junto con el ID de la boleta generada.
  • 400 Bad Request: Si no hay suficiente stock para uno de los productos especificados en items en la bodega especificada. Retorna un mensaje indicando el problema específico.
  • 500 Internal Server Error: Error interno del servidor. Retorna un mensaje indicando el error.
